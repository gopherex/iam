# RFC 9207 — OAuth 2.0 Authorization Server Issuer Identification

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc9207 · Conf: SHOULD

## Назначение
AS добавляет `iss` в authorization response → защита от mix-up атак (клиент с несколькими IdP).

## Где в API
- §23 `/oauth2/authorize` redirect-ответ

## Conformance
- Добавлять `iss` в query callback'а. Клиент сверяет с ожидаемым issuer.

## Gotchas
- Дёшево и важно при множественных провайдерах — включать по умолчанию.
