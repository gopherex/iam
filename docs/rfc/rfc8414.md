# RFC 8414 — OAuth 2.0 Authorization Server Metadata

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc8414 · Conf: MUST

## Назначение
Discovery-документ AS: `/.well-known/oauth-authorization-server` со списком endpoint'ов, поддерживаемых grant/scope/alg.

## Где в API
- §2 `/.well-known/openid-configuration` (OIDC-надмножество) + oauth-authorization-server

## Conformance
- Публиковать: `issuer, authorization_endpoint, token_endpoint, jwks_uri, revocation_endpoint, introspection_endpoint, device_authorization_endpoint, pushed_authorization_request_endpoint, code_challenge_methods_supported:["S256"], grant_types_supported`.

## Gotchas
- `issuer` должен точно совпадать с `iss` в токенах и base URL (9207).
