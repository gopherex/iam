# RFC 9068 — JWT Profile for OAuth 2.0 Access Tokens

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc9068 · Conf: MUST

## Назначение
Стандартные claim'ы access-token JWT: `iss, sub, aud, exp, iat, jti, client_id, scope, auth_time, acr, amr`. `typ:"at+jwt"`.

## Где в API
- §22 token runtime, §33 token-profiles (claims_template), §0 access token

## Conformance
- header `typ:"at+jwt"` — отличать access от id_token.
- aud = resource/audience из token-profile.
- Кастомные claim'ы — в namespaced ключах, не перетирать стандартные.

## Gotchas
- acr/amr (§MFA): отражать AAL и методы (pwd, otp, webauthn) — потребитель проверяет step-up.
