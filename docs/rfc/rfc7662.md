# RFC 7662 — OAuth 2.0 Token Introspection

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc7662 · Conf: MUST

## Назначение
Resource server спрашивает у AS состояние токена: `POST /introspect` → `{active, sub, scope, exp, ...}`.

## Где в API
- §23 `/oauth2/introspect`
- §22 `/v1/tokens/introspect` (Service/Admin)

## Conformance
- Endpoint защищён (client auth или service token) — иначе oracle для перебора.
- `active:false` минимум, без лишних claim для неактивных.

## Gotchas
- Для revocation-aware проверки: gateway зовёт introspect вместо локальной JWT-валидации, когда мгновенный отзыв критичен (см. §22 «когда»).
