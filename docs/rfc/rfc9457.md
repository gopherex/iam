# RFC 9457 — Problem Details for HTTP APIs

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc9457 · Conf: MUST (заменяет 7807)

## Назначение
Машиночитаемый формат ошибок: `{type, title, status, detail, instance}` + расширения.

## Где в API
- §0 Error envelope

## Conformance
- Наш envelope `{error:{code,message,request_id,details?}}` — совместимый профиль. `code` ≈ `type` (стабильный идентификатор).
- Content-Type `application/problem+json` (или задокументировать наш media type).

## Gotchas
- `code` — стабильная строка для SDK; не менять между версиями. HTTP-status согласован с классом.
