# NIST SP 800-63B — Digital Identity (Authentication)

- NIST · https://pages.nist.gov/800-63-3/sp800-63b.html · Conf: SHOULD

## Назначение
Современные требования к паролям/аутентификаторам: длина вместо сложности, breach-check, no forced rotation, AAL-уровни.

## Где в API
- §6 password (policy/check), §34 password-policy/mfa-policy, AAL (§10)

## Conformance
- min length >=8 (рекоменд. 12+), без обязательной композиции спецсимволов.
- Проверка по breach-листам (HIBP k-anonymity) — `breached_check` (§34).
- Не форсить периодическую смену без причины. zxcvbn score (§34 `zxcvbn_min_score`).
- AAL1/2/3 ↔ наши aal + amr в токене (9068).

## Gotchas
- HIBP — k-anonymity (первые 5 hex SHA-1), не слать пароль целиком.
