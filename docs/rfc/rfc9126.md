# RFC 9126 — Pushed Authorization Requests (PAR)

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc9126 · Conf: OPT

## Назначение
Клиент пушит authorization-параметры на backend (`POST /par`) → `request_uri`, затем `/authorize?request_uri=...`. Параметры не идут через браузер.

## Где в API
- §23 `/oauth2/par`

## Conformance
- Ответ `{request_uri, expires_in}`. request_uri одноразовый, short-lived.
- Если PAR обязателен для клиента — `require_pushed_authorization_requests:true` в его конфиге.

## Gotchas
- Нужен для high-security (FAPI) клиентов; для обычных SPA избыточен.
