# RFC 7009 — OAuth 2.0 Token Revocation

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc7009 · Conf: MUST

## Назначение
Endpoint отзыва токена клиентом: `POST /revoke` с `token` + `token_type_hint`.

## Где в API
- §23 `/oauth2/revoke`
- §22 `/v1/tokens/revoke` (внутренний, шире — по session_id)

## Conformance
- Отзыв refresh → отзывает связанные access (рекомендация).
- Всегда 200, даже если токен неизвестен (не раскрывать существование).

## Gotchas
- JWT access stateless — мгновенный отзыв требует introspection/blacklist (см. 7662). Документировать gap между revoke и истечением access TTL.
