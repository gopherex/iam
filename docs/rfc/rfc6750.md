# RFC 6750 — OAuth 2.0 Bearer Token Usage

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc6750 · Conf: MUST

## Назначение
Передача access token: `Authorization: Bearer <token>`, ответы `WWW-Authenticate` с `error=invalid_token|insufficient_scope`.

## Где в API
- §0 Auth modes (User/Admin/Service/SCIM = Bearer)
- Все protected ручки

## Conformance
- Bearer только в `Authorization` header. НЕ принимать токен из query (`access_token=`) — утечки в логи.
- 401 + `WWW-Authenticate: Bearer error="invalid_token"` при невалидном; 403 `insufficient_scope` при нехватке scope.

## Gotchas
- Cookie-mode (httpOnly) — это не Bearer; для него CSRF-токен (§0), не WWW-Authenticate.
