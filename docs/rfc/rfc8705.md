# RFC 8705 — OAuth 2.0 mTLS Client Auth & Certificate-Bound Tokens

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc8705 · Conf: OPT

## Назначение
Client auth через клиентский TLS-сертификат + привязка токена к сертификату (sender-constrained).

## Где в API
- §25 service accounts, §32 confidential clients (high-security)

## Conformance
- `tls_client_auth` / `self_signed_tls_client_auth`. Токен несёт `cnf.x5t#S256`.

## Gotchas
- Требует терминации mTLS на gateway + прокидывания cert thumbprint. Enterprise-only.
- Альтернатива sender-constrained без mTLS — DPoP (9449).
