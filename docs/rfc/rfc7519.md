# RFC 7519 — JSON Web Token (JWT)

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc7519 · Conf: MUST

## Назначение
Формат токена: header.payload.signature, registered claims `iss,sub,aud,exp,nbf,iat,jti`.

## Где в API
- §22 access/id tokens, §33 token-profiles, §0 токены

## Conformance
- Всегда проверять exp/nbf/iss/aud. jti для anti-replay где нужно.
- alg фиксировать (см. 7518/8037), запрещать `none`.

## Gotchas
- `alg:none` и confusion HS↔RS — классические дыры. Принимать только whitelisted alg из JWKS.
