# RFC 9110 — HTTP Semantics

- Status: Internet Standard (STD 97) · https://www.rfc-editor.org/rfc/rfc9110 · Conf: MUST

## Назначение
Базовая семантика HTTP: методы, статусы, conditional requests (ETag), Authorization, content negotiation.

## Где в API
- Все ручки

## Conformance
- Идемпотентность PUT/DELETE; правильные коды (201 create, 204 no-content, 409 conflict).
- ETag/If-Match для optimistic concurrency на PATCH config (опц.).

## Gotchas
- Корректный 401 vs 403 (аутентификация vs авторизация) — частая путаница.
