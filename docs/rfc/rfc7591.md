# RFC 7591 — OAuth 2.0 Dynamic Client Registration

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc7591 · Conf: OPT

## Назначение
Программная регистрация OAuth-клиентов: `POST /register` → client_id/secret.

## Где в API
- §32 admin apps (альтернатива — admin-only создание)

## Conformance
- Если открыт публично — защитить initial access token, иначе спам клиентов.

## Gotchas
- Для headless обычно достаточно admin API (§32). DCR нужен для self-service developer-портала.
