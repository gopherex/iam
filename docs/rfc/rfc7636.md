# RFC 7636 — PKCE

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc7636 · Conf: MUST

## Назначение
Proof Key for Code Exchange: защита authorization_code от перехвата. `code_challenge` (S256) на старте, `code_verifier` на обмене.

## Где в API
- §23 `/oauth2/authorize` (code_challenge), `/oauth2/token`
- §8 oauth start (code_challenge) / exchange (code_verifier)
- §3 `token/exchange` — PKCE распространён на ВСЕ browser-flow (magic/email/sso), не только OAuth

## Conformance
- method `S256` обязателен; `plain` запрещён.
- Public clients — PKCE обязателен (9700). Confidential — рекомендован.
- code_verifier: 43–128 символов [A-Z a-z 0-9 -._~].

## Gotchas
- Привязка challenge↔verifier хранится с auth_code, проверяется один раз, code одноразовый.
