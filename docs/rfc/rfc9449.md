# RFC 9449 — DPoP (Demonstrating Proof of Possession)

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc9449 · Conf: OPT

## Назначение
Sender-constrained токены для public clients (SPA/native): клиент подписывает каждый запрос proof-JWT, токен привязан к ключу (`cnf.jkt`).

## Где в API
- §3 sign-in/refresh (выдача DPoP-bound токенов), §22 introspection (cnf)

## Conformance
- Header `DPoP: <proof jwt>`; access token type `DPoP` (не Bearer).
- Проверять htu/htm/iat/jti, привязку jkt.

## Gotchas
- Защищает от кражи токена из браузера (XSS) лучше, чем httpOnly-cookie в чистом виде. Опционально включать для SPA.
