# RFC 7523 — JWT Profile for OAuth 2.0 Client Auth and Grants

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc7523 · Conf: SHOULD

## Назначение
`private_key_jwt` client authentication + `urn:ietf:params:oauth:grant-type:jwt-bearer`. Клиент подписывает JWT приватным ключом вместо client_secret.

## Где в API
- §25 service accounts (machine creds без shared secret)
- §23 token endpoint client auth

## Conformance
- client_assertion_type=`...jwt-bearer`, client_assertion=<signed JWT>.
- Проверять aud=token endpoint, exp, jti (anti-replay).

## Gotchas
- Безопаснее client_secret (нет shared secret в сети). Для S2S — предпочтительно.
