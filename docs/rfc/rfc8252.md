# RFC 8252 — OAuth 2.0 for Native Apps

- Status: BCP 212 · https://www.rfc-editor.org/rfc/rfc8252 · Conf: SHOULD

## Назначение
Best practices для native/mobile/CLI: system browser (не webview), loopback/custom-scheme redirect, обязательный PKCE.

## Где в API
- §32 app type `native`
- §8 OAuth flow native
- §24 device flow (альтернатива для CLI/TV)

## Conformance
- redirect_uri для native: `http://127.0.0.1:{port}` (loopback) или `app.scheme:/...` (custom scheme) или claimed https.
- PKCE обязателен для native.

## Gotchas
- Loopback: разрешить любой порт при exact host match — IdP не знает порт заранее.
- Embedded webview — запрещать (фишинг, нет SSO).
