# RFC 9700 — Best Current Practice for OAuth 2.0 Security

- Status: BCP · https://www.rfc-editor.org/rfc/rfc9700 · Conf: MUST

## Назначение
Актуальный security-baseline OAuth (заменяет/дополняет 6819). Что обязательно, что запрещено.

## Где в API
- §23, §8, §37 — весь OAuth/OIDC

## Conformance (обязательно)
- PKCE для ВСЕХ клиентов (не только public).
- Точное сравнение redirect_uri, без wildcard.
- Запрет implicit grant и ROPC password grant.
- Sender-constrained refresh ИЛИ refresh rotation + reuse-detection (§34 session-policy).
- `iss` в ответе (9207), mix-up защита.

## Gotchas
- Refresh reuse detection — при повторном использовании отозванного refresh убивать всю цепочку сессий.
