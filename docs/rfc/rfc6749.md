# RFC 6749 — OAuth 2.0 Authorization Framework

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc6749 · Conf: MUST

## Назначение
База провайдер-режима: authorization endpoint, token endpoint, grant types (authorization_code, client_credentials, refresh_token), client types (confidential/public), scopes, error responses.

## Где в API
- §23 `/oauth2/authorize`, `/oauth2/token` — провайдер
- §8 social — сервер как OAuth client
- §25 service accounts — client_credentials
- §32 app clients (confidential/public, redirect_uris)

## Conformance
- authorization_code grant — обязателен. Implicit grant НЕ реализуем (запрещён 9700).
- redirect_uri — точное сравнение строк, без wildcard.
- state — обязателен у клиента (CSRF), сервер пробрасывает as-is.
- Public clients — только с PKCE (7636).

## Gotchas
- Не использовать ROPC (password grant) для third-party. Core password login идёт через §3, не через `/oauth2/token`.
- error codes (`invalid_request`, `invalid_grant`, ...) на token endpoint — строго по §5.2.
