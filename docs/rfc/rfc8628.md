# RFC 8628 — OAuth 2.0 Device Authorization Grant

- Status: Proposed Standard · https://www.rfc-editor.org/rfc/rfc8628 · Conf: MUST (для CLI/TV)

## Назначение
Device flow: устройство без браузера получает `device_code`+`user_code`, юзер аппрувит на другом устройстве, device поллит token endpoint.

## Где в API
- §23 `/oauth2/device_authorization`
- §24 `/v1/device`, `/device/approve`, `/device/deny` (headless: страницу рендерит продукт)

## Conformance
- `verification_uri` → device-страница ПРОДУКТА (headless), не UI сервера.
- token endpoint: grant `urn:ietf:params:oauth:grant-type:device_code`; ответы `authorization_pending`, `slow_down`, `expired_token`, `access_denied`.
- Соблюдать `interval` поллинга; `slow_down` +5s.

## Gotchas
- user_code: короткий, человекочитаемый (BCDFGHJKLMNPQRSTVWXZ), без похожих символов.
- verification_uri_complete (user_code в URL) для QR.
