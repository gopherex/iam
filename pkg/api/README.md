# pkg/api — generated IAM API

Generated by [ogen](https://ogen.dev) from `openapi/openapi.yaml`. **Do not edit
by hand** — run `make generate-go` to regenerate. Files are committed so
importers need no codegen toolchain.

All ogen features are enabled (see [`../../.ogen.yaml`](../../.ogen.yaml)):
client + server, request/response validation, request options & editors,
reentrant security, OpenTelemetry instrumentation, fakers and example tests.

## OpenAPI 3.0 projection

ogen consumes OpenAPI 3.0; the source spec is strict 3.1. `make generate-go`
first runs `scripts/openapi_to_30.py` to down-project (`type: [T,"null"]` →
`type: T` + `nullable`) into `openapi/.build/openapi.3.0.yaml` (a build
artifact), then runs ogen on it. The 3.1 spec stays the source of truth.

## Coverage

All 280 operations are generated (`ignore_not_implemented` is empty). Union
responses use `oneOf` + a `result_type` discriminator, so ogen emits a typed
sum type (switch on `.Type`):

| Response schema | Variants (`result_type`) |
| --- | --- |
| `AuthResultOrNextStep` (sign-in) | `authenticated`, `next_step` |
| `StepUpResult` | `next_step`, `ok` |
| `PhoneVerifyResult` | `authenticated`, `user_updated` |

The OAuth `PushedAuthorizationRequest` (PAR) form body is typed (RFC 9126), so
the form endpoint generates too.
